# Git Push Guide | Інструкція з оновлення репозиторію

Цей файл містить покрокові інструкції для фіксації змін та відправки коду на GitHub.

## 1. Базовий цикл оновлення (Standard Workflow)

Кожного разу, коли ти вносиш зміни в код (JSON, TypeScript, сервіси), виконуй ці команди в терміналі:

```bash
# 1. Перевірити статус (які файли змінено)
git status

# 2. Додати всі зміни до індексу
git add .

# 3. Створити комміт (зберегти зміни локально)
git commit -m "Опис змін: додав 10 стел та логіку плитки"

# 4. Відправити зміни на сервер
git push origin main




2. Якщо виникає помилка (Conflict/Out of date)

Якщо інший пристрій вносив зміни, або GitHub відхиляє push, спочатку затягни свіжі дані:
Bash

# Завантажити оновлення з сервера та об'єднати
git pull origin main

# Після цього повтори push
git push origin main


3. Робота з папками

Git не бачить порожні папки. Щоб папка з'явилася на GitHub:

    Створи папку (наприклад, models/stele/).

    Додай туди хоча б один файл (наприклад, .glb або пустий .gitkeep).

    Виконай стандартний git add . та git push.

4. Корисні команди

    git log --oneline — перегляд історії останніх змін (коротко).

    git checkout . — УВАГА: скасувати всі незакріплені зміни (повернути до стану останнього комміту).

    git remote -v — перевірити, до якого репозиторію підключено проект.

5. Поради щодо коммітів

    Пиши зрозумілі назви: fix: repair tile seam logic замість update 1.

    Використовуй префікси: feat: (нова фіча), fix: (виправлення), docs: (документація).
